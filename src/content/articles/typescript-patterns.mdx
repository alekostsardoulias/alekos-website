# TypeScript Patterns I Use in Every Project

TypeScript's type system is expressive enough to encode business rules directly into your code. Over years of working with it across frontend and backend projects, a handful of patterns have become automatic for me — patterns that eliminate entire categories of bugs before they reach runtime.

## Branded Types for Domain Safety

Plain strings and numbers are ambiguous. A `userId` and an `orderId` are both strings, but passing one where the other is expected should be a compile-time error. Branded types — also called opaque types — solve this with zero runtime cost:

```ts
type UserId = string & { readonly __brand: 'UserId' };
type OrderId = string & { readonly __brand: 'OrderId' };
```

Now your function signatures document intent, and TypeScript prevents you from accidentally swapping IDs. I add these to every project within the first hour of development. The return on investment is immediate.

## Discriminated Unions Over Optional Fields

When a piece of state can be in multiple shapes — loading, loaded, error — reach for a discriminated union instead of optional fields and boolean flags:

```ts
type State =
  | { status: 'idle' }
  | { status: 'loading' }
  | { status: 'loaded'; data: Data }
  | { status: 'error'; message: string };
```

This pattern makes impossible states impossible. You cannot access `data` when the status is `loading` because TypeScript will not let you. Every conditional that checks the discriminator narrows the type automatically.

## Exhaustive Type Narrowing

For any union type that represents a finite set of cases, an exhaustive check in a utility function catches missing branches at compile time when you add a new variant. Combined with discriminated unions, this gives you a poor man's pattern matching that is surprisingly powerful.

These patterns are not clever tricks — they are discipline. Once you adopt them, going back to untyped or loosely typed code feels like driving without a seatbelt. You notice how many runtime errors TypeScript was silently preventing.
